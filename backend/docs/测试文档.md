## 完整测试流程

### 场景1：管理员完整流程

#### 步骤1：管理员登录

```bash
curl -X POST http://localhost:8000/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{
    "username": "admin",
    "password": "admin123"
  }'
```

**预期响应**:
```json
{
  "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "token_type": "bearer",
  "user": {
    "role": "admin",
    "auth_type": "admin"
  }
}
```

**保存token**: 将 `access_token` 保存到环境变量
```bash
export TOKEN="eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
```

### 场景2：GitHub 用户流程

#### 步骤1：获取GitHub授权URL

```bash
curl -X GET http://localhost:8000/api/auth/github/login
```

**预期响应**:
```json

  "auth_url": "https://github.com/login/oauth/authorize?client_id=Iv23limIJjAfoyycBZvr&redirect_uri=http://localhost:8000/api/auth/github/callback&scope=read:user user:email"
}
```

---

#### 步骤2：用户授权

1. 在浏览器中打开上面返回的 `auth_url`
2. 登录 GitHub 并授权
3. GitHub 会重定向到 `http://localhost:8000/api/auth/github/callback?code=xxx`
4. 后端会返回 JWT token

**手动测试回调**（使用测试code）:
```bash
# 注意：这个 code 需要从 GitHub 实际授权后获取
curl -X GET "http://localhost:8000/api/auth/github/callback?code=YOUR_GITHUB_CODE"
```

**预期响应**:
```json
{
  "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "token_type": "bearer",
  "user": {
    "role": "user",
    "auth_type": "github",
    "github_id": 12345678,
    "github_username": "octocat"
  }
}
```

**保存token**:
```bash
export GITHUB_TOKEN="eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
```

## 自动化测试脚本

创建一个测试脚本 `test.sh`:

```bash
#!/bin/bash

BASE_URL="http://localhost:8000"

echo "=== 测试开始 ==="

# 1. 健康检查
echo "\n1. 健康检查"
curl -s $BASE_URL/health | jq

# 2. 管理员登录
echo "\n2. 管理员登录"
LOGIN_RESPONSE=$(curl -s -X POST $BASE_URL/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"admin123"}')
echo $LOGIN_RESPONSE | jq

TOKEN=$(echo $LOGIN_RESPONSE | jq -r '.access_token')
echo "Token: $TOKEN"

# 3. 创建用户
echo "\n3. 创建用户"
curl -s -X POST $BASE_URL/api/users \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $TOKEN" \
  -d '{"username":"testuser"}' | jq

# 4. 绑定用户名
echo "\n4. 绑定用户名"
curl -s -X POST $BASE_URL/api/auth/bind-username \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $TOKEN" \
  -d '{"username":"testuser"}' | jq

# 5. 创建项目
echo "\n5. 创建项目"
PROJECT_RESPONSE=$(curl -s -X POST $BASE_URL/api/projects/testuser \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $TOKEN" \
  -d '{"name":"测试项目","description":"自动化测试"}')
echo $PROJECT_RESPONSE | jq

PROJECT_ID=$(echo $PROJECT_RESPONSE | jq -r '.data.id')
echo "Project ID: $PROJECT_ID"

# 6. 获取项目列表
echo "\n6. 获取项目列表"
curl -s -X GET $BASE_URL/api/projects/testuser | jq

# 7. 更新项目
echo "\n7. 更新项目"
curl -s -X PUT $BASE_URL/api/projects/testuser/$PROJECT_ID \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $TOKEN" \
  -d '{"name":"更新后的项目"}' | jq

# 8. 删除项目
echo "\n8. 删除项目"
curl -s -X DELETE $BASE_URL/api/projects/testuser/$PROJECT_ID \
  -H "Authorization: Bearer $TOKEN" | jq

# 9. 删除用户
echo "\n9. 删除用户"
curl -s -X DELETE $BASE_URL/api/users/testuser \
  -H "Authorization: Bearer $TOKEN" | jq

echo "\n=== 测试完成 ==="
```

**运行测试**:
```bash
chmod +x test.sh
./test.sh
```

## 注意事项

1. **GitHub OAuth 测试**
   - 需要实际的 GitHub 账号进行授权
   - callback 地址需要在 GitHub App 设置中配置
   - 测试环境建议使用 ngrok 等工具暴露本地服务

2. **Token 过期**
   - Token 默认有效期 24 小时
   - 过期后需要重新登录获取新token

3. **数据持久化**
   - 所有数据存储在 `data/users/` 目录
   - 删除目录可以清空所有数据

4. **权限测试**
   - 确保测试权限边界场景
   - GitHub 用户只能操作绑定的用户数据
   - 管理员可以操作绑定的用户数据

5. **并发测试**
   - 可以使用 Apache Bench (ab) 或 wrk 进行压力测试
   - 示例: `ab -n 1000 -c 10 http://localhost:8000/health`

## 常见问题

**Q: GitHub 回调失败？**
A: 检查 config.yaml 中的 redirect_uri 是否与 GitHub App 设置一致

**Q: 绑定用户名失败？**
A: 确保用户已经创建（使用管理员创建用户）

**Q: Token 无效？**
A: 检查 token 是否正确复制，是否过期

**Q: 权限错误？**
A: 确认已绑定用户名，且操作的是绑定的用户数据
